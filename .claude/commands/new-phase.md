---
description: 進入下個 Phase 的 checklist
---

執行以下步驟：

1. 讀 `DEVELOPMENT_PLAN.md`，找出：
   - 目前在哪個 Phase
   - 前一個 Phase 的驗收條件是否已滿足
2. 若前一個 Phase 未完成，列出未勾選的任務並**暫停**，詢問是否要先補完
3. 若已完成：
   a. 列出下個 Phase 的所有任務
   b. 標示哪幾項會需要使用者介入（下載模型、提供測試影片、手動驗證等）
   c. 建議一個合理的開始順序
   d. 問使用者要從哪項開始、是否要建立新的 git branch（格式建議：`phase-N-short-name`）
4. **不要自動建立 branch 或自動開始實作**，等使用者確認
