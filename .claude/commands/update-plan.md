description: 依當前進度更新 DEVELOPMENT_PLAN.md 勾選狀態
---

執行以下步驟：

1. 讀 `DEVELOPMENT_PLAN.md`，找到當前 Phase
2. 回顧本次對話（或最近的 git 變動）做了什麼
3. 將對應任務的 `[ ]` 改為 `[x]`
4. 若做了原本沒規劃的任務，在對應 Phase 加一行並標記為完成（格式：`- [x] X.Y 新任務描述（本次新增）`）
5. 若發現規劃需要**結構性調整**（新增子 Phase、調整順序），**先停下來問使用者**，不要擅自改動
6. 改完後回報：改了哪幾項、是否有需要使用者決策的地方

**重要**：這個指令只做進度同步，不重寫計畫內容。計畫內容的大改動要獨立討論。
